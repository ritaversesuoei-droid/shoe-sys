# 改善基準告示 閾値 確認資料（社労士確認用）

**対象**: 昭栄運輸 運行・勤怠管理システム
**規程**: 自動車運転者の労働時間等の改善のための基準（改善基準告示）／**トラック運転者**／2024年4月1日適用
**設定バージョン**: `2024-04`（`app_settings('compliance')`）
**作成**: 開発担当（AI実装による暫定値）　**確認依頼先**: 社会保険労務士

> ⚠ **本資料の閾値はAIが2024年改善基準告示の一般値に基づいて設定した暫定値です。**
> 各値の妥当性・特例の扱いについて、社労士のご確認をお願いします。
> 確認・補正の結果は設定（`app_settings('compliance')`）を更新するだけでシステムへ即反映できます（コード改修不要）。

---

## 0. 確認のしかた（要点）

- 下表の各行について、**「値は適正か」「自動判定の運用で良いか」**をご確認ください。
- 補正が必要な場合は「確認結果」欄に正しい値・条件をご記入ください。
- 本システムは閾値を**日本標準時(JST)・分単位**で保持します（表では時間表記を併記）。
- 確認後の補正値は、管理画面の設定または `supabase/migrations/0007` 相当の設定更新で反映し、
  既存データは再計算（`recompute`）で全勤務を再判定できます。

## 2. 各項目の詳細・確認ポイント

### 2.1 1日の拘束時間（#1〜#3） ✅自動判定
- **挙動**: 1勤務の拘束=確定退勤−確定出勤。
  - 15時間超 → **違反**
  - 14時間超〜15時間以下 → 同一週(月〜日)で**3回目以降は違反**、2回目までは**警告**
- **法令根拠(暫定)**: 原則13h／最大15h／14h超は週2回が目安。
- **確認ポイント**:
  - [ ] 「14時間超 週2回まで」を**上限(超過=違反)**として扱う運用で良いか（告示上は努力目標的との解釈もあり）。
  - [ ] 週の起算（本システムは**月曜起算**）で良いか。

### 2.2 休息期間（#4・#5） ✅自動判定
- **挙動**: 前勤務の退勤→当勤務の出勤の間隔。
  - 9時間未満 → **違反**／ 9時間以上11時間未満 → **警告**
- **法令根拠(暫定)**: 継続11hを基本、9hを下回らない。
- **確認ポイント**:
  - [ ] 下限9h・基本11hで良いか。
  - [ ] **分割休息の特例**（合計10h以上・1回3h以上 等）を考慮する必要があるか（§4参照）。

### 2.3 1か月／1年の拘束時間（#6〜#9） ⚠未実装（値のみ設定）
- **法令根拠(暫定)**: 月284h（協定310h）／年3,300h（協定3,400h）。
- **確認ポイント**:
  - [ ] 値（284/310/3300/3400）で良いか。
  - [ ] **複合条件**の要否: 「月284h超は**連続3か月まで**」「協定310hは**年6か月まで**」
        「月の時間外・休日労働が**100時間未満**となるよう努める」等を判定対象とするか。

### 2.4 運転時間（#10・#11） ⚠未実装（値のみ設定）
- **法令根拠(暫定)**: 2日（始業から48h）平均で1日9h以内、2週平均で1週44h以内。
- **確認ポイント**:
  - [ ] 「2日平均」の起算（始業時刻起算48h）の解釈で良いか。
  - [ ] **運転時間そのものの取得方法**: 現在は出発〜退勤の拘束/労働は把握できるが、
        純粋な「運転時間」（待機・荷役を除く）を打刻から厳密に取得できない。
        運転時間の算定方法（みなし／別途記録）をご教示いただきたい。

### 2.5 連続運転時間（#12・#13） ⚠未実装（値のみ設定）
- **法令根拠(暫定)**: 連続運転4h以内、中断は1回連続10分以上・合計30分以上。
- **確認ポイント**:
  - [ ] 中断（休憩・荷役等）の判定に用いる打刻（到着/荷卸/休憩）の対応で良いか。

### 2.6 深夜帯・所定労働（#14・#15） 参考
- 深夜帯 22:00–05:00 は**労基法37条の割増対象**の集計に使用（改善基準告示の閾値ではない）。
- 所定労働 8h/日 は**残業（労働−所定）算定の基準**（`app_settings('payroll')`）。
  - [ ] 事業所の所定労働時間（8時間で良いか）。

| 自動判定が**実装済み** | 自動判定は**未実装**（値は設定済み・手動確認運用） |

## 4. 未モデル化の特例（適用要否のご確認をお願いします）

以下は2024改善基準告示の特例・付随ルールで、現在システムに**モデル化されていません**。
御社の運行実態に照らして適用要否をご教示ください（適用する場合は実装します）。

- [ ] **分割休息特例**: 休息を分割付与（合計10h以上、1回3h以上、2〜3分割 等）。
- [ ] **2人乗務特例**: 拘束を20hまで延長／休息を4hまで短縮可。
- [ ] **フェリー特例**: フェリー乗船時間を休息期間として扱う（原則）。
- [ ] **休日労働**: 2週間に1回以内、かつ休日労働で拘束時間の上限を超えない。
- [ ] **隔日勤務 等**（トラックでは通常非該当か）。
- [ ] **36協定との関係**: 時間外・休日労働の月/年上限の扱い。

## 6. 反映手順（確認後・開発側作業）

1. 確認結果の値を `app_settings('compliance')` に反映（管理画面の設定 or `supabase/migrations/0007` 更新）。
2. §3「未実装」項目で判定対象が確定したものは、判定ロジックを追加実装（`src/lib/compliance/`）。
3. 既存の全勤務を再計算（`recompute`）して、新基準で違反台帳を再構築。
4. 月次・違反画面で突合確認。
